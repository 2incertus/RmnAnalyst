Deployment Proposal: RMN Analyst

Executive Summary

The goal is to deploy the RMN Analyst application for internal use, accessible to team members via the company VPN. To ensure a secure, scalable, and maintainable deployment, we propose containerizing the application using Docker.

This approach packages the application and its environment into a single, portable artifact. It standardizes the deployment process, simplifying the work for the engineering team and guaranteeing consistency from development to production.

Proposed Solution: Containerization with Docker

We will create a Docker image that contains the fully built, production-ready static web application and a lightweight web server.

Key Benefits:

  • Portability: Runs consistently on any machine with Docker installed, from a developer'''s laptop to a production server.
  • Isolation: The application and its dependencies are isolated, preventing conflicts with other services on the host machine.
  • Efficiency: The final image is minimal, containing only what is necessary to serve the application, resulting in a smaller footprint and faster start times.
  • Simplified Hand-off: The engineering team receives a standard, self-contained Docker image, not a set of source files and build instructions.

Implementation Plan

The following files will be created in the project root to facilitate this process:

  1. `serve` dependency: A lightweight, production-grade static file server will be added to the project.
  2. `package.json` script: A `start` script will be added to run the production server.
  3. `.dockerignore` file: To exclude unnecessary files (like `node_modules`, `.git`) from the Docker image, making it smaller and faster to build.
  4. `Dockerfile`: A blueprint for building the application image. It will define a multi-stage build to keep the final image lean:
    • Build Stage: Installs dependencies, compiles the TypeScript/React code via `npm run build`.
    • Serve Stage: Copies the compiled assets from the build stage into a minimal environment with the production web server.

Instructions for Engineering Team

Once the Dockerfile is in place, the deployment process will be straightforward:

1. Build the Docker Image

Navigate to the project root and run the following command. This will build the image and tag it as rmn-analyst.

docker build -t rmn-analyst .

2. Run the Docker Container

Run the image as a container. This command starts the container in detached mode and maps the container'''s internal port 80 to port 3000 on the host machine.

docker run -d -p 3000:80 rmn-analyst

3. Access the Application

The RMN Analyst application will then be available at http://<server-ip>:3000 for anyone on the VPN.

Conclusion

This Docker-based deployment strategy is a modern, standard, and robust solution for hosting internal web applications. It provides a clear and simple path for the engineering team to deploy and manage the RMN Analyst tool, ensuring a smooth and reliable experience for all users.
